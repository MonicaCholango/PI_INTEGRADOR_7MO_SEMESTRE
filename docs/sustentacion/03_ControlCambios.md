# Control de Cambios y CCB

## Proceso de Control de Cambios

### Flujo Establecido:
1. **Solicitud (RFC)** - Issue en GitHub con template
2. **Evaluación** - Análisis de impacto técnico  
3. **Aprobación** - Decisión del líder de proyecto
4. **Implementación** - Desarrollo en rama dedicada
5. **Verificación** - Testing y revisión de código
6. **Cierre** - Merge y actualización de baseline
